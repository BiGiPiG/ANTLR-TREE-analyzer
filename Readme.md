## ANTLR+TREE analyzer

<p>
Проект представляет собой синтаксический и семантический анализатор для формального языка, описывающего выражения с операциями присваивания (=) и инкремента (++) над целочисленными литералами (INT) и идентификаторами переменных (VAR). Язык относится к типу 3 по иерархии Хомского (регулярные языки) и обладает однозначной LL(1)/SLR(1)-грамматикой.
Ключевые особенности языка:
Поддержка вложенных цепочек присваиваний: a = b = ++c
Префиксный, постфиксный и каскадный инкремент: ++x, y++, ++++z
Строгое соответствие регулярному выражению: (++*(VAR|INT)++*)(=++*(VAR|INT)++*)*
Отсутствие пробелов, арифметических, логических операций и иных терминалов.
</p>


## Инструкция по сборке и запуску проекта ANTLR+TREE

### 1. Предварительные требования
- Система контроля версий Git
- Python 3.8 или выше
- Java Development Kit (JDK) 11 или выше
- Менеджер пакетов pip

### 2. Клонирование репозитория
Откройте терминал и выполните:
```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd <ИМЯ_ПАПКИ_ПРОЕКТА
```

### 3. Настройка виртуального окружения
```bash
# Создание изолированного окружения
python -m venv .venv

# Активация (Windows PowerShell/CMD)
.venv\Scripts\activate
# Активация (Linux/macOS)
# source .venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 4. Генерация лексера и парсера
Убедитесь, что файл antlr-4.13.2-complete.jar находится в корневой директории проекта. Затем выполните:
```bash
java -jar antlr-4.13.2-complete.jar -Dlanguage=Python3 -visitor Transliatory.g4
```
>Команда создаст файлы TransliatoryLexer.py, TransliatoryParser.py, TransliatoryVisitor.py и сопутствующие .tokens.

### 5.1 Запуск проекта (локально)

Анализ отдельного файла
```bash
python analyzer.py examples/valid/valid_sequential.txt
```

Пакетное тестирование
```bash
python run_tests.py
```
>Скрипт автоматически обработает все файлы в директории examples/, выведет деревья разбора, 
> результаты семантической проверки и итоговую символическую таблицу для каждого набора.

### 5.2 Запуск через Docker

#### Требования:
- Установленные Docker и Docker Compose
- Файл antlr-4.13.2-complete.jar должен располагаться в корне проекта
- В корне проекта должны присутствовать файлы Dockerfile и docker-compose.yml

#### Инструкция:
1. Соберите образ и запустите контейнер:
```bash
docker compose up --build
```
> Система автоматически подготовит среду (Python + Java), 
> сгенерирует файлы лексера/парсера из грамматики и запустит скрипт run_tests.py.

2. Анализ отдельного файла через Docker
```bash
docker compose run --rm antlr-analyzer python analyzer.py examples/valid/valid_sequential.txt
```
> В docker-compose.yml директория examples/ подключена как volume. 
> Это позволяет редактировать или добавлять тестовые файлы на хост-машине и сразу проверять 
> их через Docker без пересборки образа.
